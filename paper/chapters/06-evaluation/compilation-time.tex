\section{Compilation Time}\label{sec:compile-time}

Table~\ref{tab:benchmark-wall-clock} reports end-to-end wall-clock means for the three benchmark programs on version~1.0
and version~2.0.
All pairs produced byte-identical MIDI output (Table~\ref{tab:benchmark-midi-size}), so the timing difference isolates
compiler implementation changes rather than workload variation.
Version~2.0 adds between $-0.01$~ms and $+0.14$~ms relative to version~1.0, or roughly $0$--$2\%$ overhead on these
workloads.

On the benchmark machine, the largest fixtures finish in about 8~ms end-to-end (Table~\ref{tab:benchmark-wall-clock}).
Hardware-specific absolute times may differ on other hosts; the relevant engineering question for Motivo Studio is
whether explicit recompilation stays within interactive latency.

Table~\ref{tab:benchmark-per-stage} breaks the in-process work down by pipeline stage.
Parsing and semantic analysis remain below $0.04$~ms regardless of program; lowering and MIDI writing dominate, which
matches the expected $O(E)$ lowering cost and $O(E \log E)$ MIDI serialization cost.
The semantic column shows a slight increase in version~2.0 (for example B1: $0.0058$~ms $\rightarrow$ $0.0067$~ms),
consistent with explicit type checking, but the absolute cost is negligible compared with lowering and backend I/O.

The near-linear growth in the single-loop case (B1) is consistent with the implementation.
Parsing and semantic analysis see a small source program, while lowering and MIDI writing scale with the number of
generated events.
B3 stresses nested loop structure rather than a flat iteration count; its lowering time exceeds B1 despite
the same event
cap because the lowering engine walks nested control-flow frames.
The backend also sorts events by tick before computing delta-times.
Cost grows with event count and nested structure (B3 exceeds B1 in lowering time at the same cap) but remains in the
low-millisecond range on the benchmark machine.

\subsection{Execution Speed and Studio Workflow}\label{detail-subsec:execution-speed-and-real-time-viability}

Motivo Studio compiles when the user presses the compile button, not on every keystroke or on save.
At roughly 5--8~ms for programs up to the 20,000-event cap on the tested Release build, that whole-file compile step is
fast enough that the editor's compile-to-preview loop feels immediate in practice.
Per-keystroke recompilation would require incremental parsing and partial lowering; the current architecture does not
implement that optimization, but the measured throughput leaves headroom if such a feature were added later.
Typical studio pieces (hundreds to a few thousand events) compile even faster than the B1--B3 stress fixtures, so
auditory and visual feedback does not depend on an interpreted runtime at playback.
