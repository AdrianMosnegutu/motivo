\section{Closing Remarks}\label{sec:closing}

Flat event editing loses algorithmic intent, while live-coding systems recover structure by coupling
composition to a running synthesizer.
Motivo sits between these approaches: structure remains in source, expansion is deterministic, and the
artifact is a Format1 MIDI file importable into any host.
On the benchmark workloads in Chapter\ref{ch:evaluation}, a handful of source lines expand into thousands of
note events, compilation remains in the low-millisecond range on a Release build, and output is byte-reproducible.
The gap between flat event editing and runtime-coupled live systems from Section~\ref{sec:motivation} is
therefore closed for offline, rule-based symbolic composition within the scope.

Motivo complements existing music-production tools where repetition and symbolic rules dominate; within that
boundary, we delivered a validating compiler, a deployable studio, and performance suitable for interactive use.

This thesis set out to determine whether a small compiled language could keep compositional structure
explicit in source and expand it deterministically into a standard file without a playback runtime.
Within that boundary, we delivered a validating compiler, a deployable studio, and evidence that the pipeline is fast
enough for interactive local use on the tested workloads.
