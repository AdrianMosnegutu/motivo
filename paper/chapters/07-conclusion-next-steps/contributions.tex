\section{Contributions}\label{sec:contributions}

Motivo demonstrates that compositional structure can stay explicit in typed source and expand deterministically into a
standard MIDI file without a language runtime at playback.
We delivered that pipeline end to end.

\topic{Language and compiler.}
Motivo maps notes, patterns, tracks, and voices to a closed type system and a staged native compiler.
Lowering treats sequential statements, parallel voices, and typed pattern calls as a temporal expansion problem rather
than as raw byte assembly.
Version~2.0 replaced implicit \texttt{let} typing with explicit declarations and signature-based overload resolution;
evaluation shows roughly $0$--$2\%$ end-to-end overhead on large fixtures with byte-identical MIDI.

\topic{Motivo Studio and engineering.}
Motivo Studio wraps \texttt{motivoc} in a Next.js editor with diagnostics, piano-roll preview, browser playback, and
authenticated file storage.
The monorepo enforces quality with automated tests, paper CI, and deployment to \texttt{motivo-studio.eu} as a
maintainable product rather than a one-off prototype.

\topic{Evaluation.}
Synthetic benchmarks compile up to 20,000 events in about 5--8~ms on a local Release build; lowering and MIDI
serialization dominate per-stage cost.
These results are consistent with the use cases introduced in Section~\ref{sec:use-cases},
though the evaluation does not yet include a user study of composer preference or learnability.
