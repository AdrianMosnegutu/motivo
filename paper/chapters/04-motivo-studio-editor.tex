\chapter{Motivo Studio: Web Editor and Client Architecture}\label{ch:studio-editor}

The compiler is the core technical artifact, but a compiler alone is not a complete user-facing system.
Motivo Studio was developed to make the language usable from a browser: the user writes Motivo source, sends it to a
backend service, receives either MIDI bytes or diagnostics, and sees the resulting music in an editor-centered
interface.
This chapter describes that \textit{product layer}---architecture, compile request flow, editor integration,
visualization, playback, authentication, and file management---without treating the application as a secondary demo.

Deployment, automated testing, and continuous integration are covered in Chapter~\ref{ch:deployment-ci}.
